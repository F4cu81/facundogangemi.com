# HOMEPAGE UI DIRECTION — facundogangemi.com

## Document Purpose

This document defines the visual UI direction for the homepage of facundogangemi.com.

It translates the approved strategic foundation, content briefs, UX architecture and wireframe specification into a concrete visual direction for future UI design and frontend development.

This document should guide the implementation of the homepage in a static website environment compatible with GitHub Pages.

The UI must feel like:

> Executive AI Premium

The homepage must communicate:

- executive authority
- AI strategy expertise
- enterprise transformation credibility
- intelligent operating model thinking
- premium advisory positioning
- modern digital sophistication

The UI must not feel like:

- a generic consulting template
- a startup landing page
- a freelancer portfolio
- a traditional Agile Coach website
- a WordPress-style blog homepage
- a sci-fi or cyberpunk AI interface

---

# 1. Core Visual Concept

## 1.1 Visual Direction

The visual identity of the homepage is:

> Executive AI Premium

This means the homepage should combine:

- dark executive backgrounds
- refined typography
- restrained AI-inspired visual details
- premium spacing
- sharp content hierarchy
- high contrast CTAs
- editorial content rhythm
- subtle motion
- enterprise-grade polish

The experience should feel modern and intelligent, but not overly futuristic.

---

## 1.2 Desired First Impression

Within the first few seconds, the visitor should feel:

- this is premium
- this is executive-level
- this person understands AI and business
- this is not generic Agile coaching
- this could help my organization think clearly about transformation

The visual perception should be closer to:

- OpenAI-level minimalism
- Stripe-level UI polish
- McKinsey Digital-level consulting sophistication
- Palantir-level enterprise seriousness
- Anthropic-level restraint and intelligence

## 2.2 Optional Alternative Accent

If the electric-blue version is chosen, use:

Electric Blue:

HEX: #129BFF

Usage:

- CTA buttons

- icon accents
- hover states

Perception:

Recommended homepage balance:

- 60% deep navy / charcoal dark surfaces
- 25% light gray / white editorial space

Avoid using the accent color for decoration without function.

# 3. Typography Direction

## 3.1 Font Pairing

Primary font:

- Inter

Secondary font:

- Manrope

Recommended usage:

- Manrope: H1, H2, major statements, KPI numerals
- Inter: body text, UI labels, navigation, buttons, captions

---

## 3.2 Typography Personality

Typography must feel:

- executive
- calm
- readable
- modern
- premium
- confident

Avoid:

- overly playful typography
- heavy decorative font choices
- aggressive startup-style oversized text
- dense corporate paragraphs

---

- 64px to 76px
- line-height: 0.95 to 1.05
- weight: 500 to 600

- 42px to 52px
- line-height: 1.05 to 1.15
- weight: 500 to 600

H3 / Card titles:

- 20px to 24px
- line-height: 1.2
- weight: 600

Body large:

- 18px to 20px
- line-height: 1.6

Body standard:

- 16px to 18px
- line-height: 1.55 to 1.7

Eyebrows / labels:

- 12px to 13px
- uppercase
- letter-spacing: 0.08em to 0.12em
- weight: 600

---

Hero H1:

- 40px to 48px
- line-height: 1.05

Section H2:

- 30px to 36px

Eyebrow:

# 4. Global UI Style

## 4.1 Overall Interface Feel

The interface should feel:

- spacious
- intentional
- structured
- clean
- premium
- editorial
- enterprise-grade

The homepage should not feel overloaded with boxes, colors or effects.

Every section should have a clear focal point.

---

## 4.2 Surface Treatment

Use three main surface types:

### Dark Premium Surface

Used for:

- hero
- KPI strip
- framework section if dark variant is selected
- assessment CTA
- final CTA
- footer

Style:

- deep navy background
- subtle radial gradient
- low-opacity line or particle detail
- high contrast white typography

---

### Light Editorial Surface

Used for:

- advisory grid
- case studies
- insights
- about section if light variant selected

Style:

- #F2F2F2 or near-white
- black/navy typography
- cards with subtle borders
- generous whitespace

---

### Elevated Card Surface

Used for:

- service cards
- case cards
- insight cards
- talk cards
- framework panels

Style:

- white or dark navy depending on section
- subtle 1px border
- soft shadow only if needed
- rounded corners: 16px to 24px
- hover state with slight lift

# 5. Section-by-Section UI Direction

---

# Section 1 — Hero

## Visual Direction

The hero should be the strongest visual moment of the homepage.

Recommended style:

- full-width deep navy background
- subtle gradient from left dark navy to right charcoal/navy
- executive portrait on the right
- abstract AI-inspired visual field behind portrait
- minimal floating quote card
- large confident headline on the left

---

## Layout Composition

Desktop:

- left content block takes approximately 48% width
- right visual block takes approximately 52% width
- content vertically centered
- H1 has strong visual dominance
- CTA row sits close to the subheadline

Mobile:

- text-first composition
- image appears below CTAs or as background crop
- hero remains readable and not too tall

---

## Hero Background

Recommended background elements:

- deep navy base
- radial gradient behind portrait
- low-opacity AI network lines
- tiny particles with low brightness
- no heavy neon effect

Avoid:

- sci-fi holograms
- excessive glowing nodes
- cyberpunk colors
- busy abstract backgrounds

---

## Hero Portrait Treatment

Portrait should feel:

- ultra-realistic
- executive
- calm
- strategic
- premium

Recommended styling:

- dark navy blazer or black suit
- strong but natural light
- clean cutout or blended background
- slight shadow integration
- no artificial beautification

The portrait should not feel AI-generated.

---

## Hero CTA Buttons

Primary button:

- solid accent background
- white text
- right arrow icon
- height: 48px to 56px
- border radius: 4px to 8px or 999px depending on final style

Secondary button:

- transparent or dark surface
- 1px border
- white text
- subtle hover fill

Recommended text:

- Book Advisory Call
- Explore Services

---

# Section 2 — Trust Bar

## Visual Direction

The trust bar should feel quiet and credible.

Recommended style:

- dark continuation from hero or slightly lighter navy band
- small uppercase label
- monochrome logos
- balanced spacing

---

## Logo Treatment

Logos should be:

- white or light gray
- equalized in visual height
- low contrast until hover
- not colorful unless required by brand constraints

Hover:

- opacity increases from approximately 0.65 to 1.0

---

# Section 3 — Strategic Advisory Grid

## Visual Direction

This section should feel like the first clear explanation of the offer.

Recommended style:

- light editorial background
- black/navy typography
- clean cards
- minimal line icons
- accent hover details

---

## Section Header Composition

Left:

- eyebrow
- H2

Right:

- short paragraph with strong clarity

This creates a premium editorial layout and avoids stacking too much text.

---

## Card Style

Service cards should use:

- white background
- 1px border in light gray/navy opacity
- rounded corners 16px to 20px
- internal padding 28px to 36px
- icon at top
- title below icon
- body text below title
- arrow link at bottom

Hover state:

- slight lift: 4px
- border color shifts to accent
- arrow moves 4px right
- icon stroke becomes accent

---

# Section 4 — Authority KPI Strip

## Visual Direction

This section should feel like a confident editorial proof band.

Recommended style:

- dark navy full-width band
- large numerals or labels in accent color
- short descriptors in white/gray
- subtle dividers between KPI blocks

Avoid:

- dashboard style
- excessive metrics
- small unreadable text

---

## KPI Typography

KPI number / label:

- Manrope
- 44px to 60px desktop
- accent color
- medium weight

KPI descriptor:

- Inter
- 14px to 16px
- white or light gray

---

# Section 5 — Signature Framework Teaser

## Visual Direction

This section should demonstrate strategic depth and proprietary thinking.

Recommended style:

Option A — Dark Premium:

- navy background
- white typography
- framework visual in accent lines
- subtle AI grid or radial glow

Option B — Light Editorial:

- light background
- dark text
- framework visual as navy/orange line system

Recommended choice:

Use dark premium if the previous KPI band can transition smoothly.

---

## Framework Visual Style

The framework should be:

- simple
- elegant
- diagrammatic
- clear
- not overly academic

Recommended visual formats:

- circular 4-phase loop
- horizontal step system
- connected node system

Recommended labels:

- Discover
- Design
- Deploy
- Evolve

Each phase should have:

- short label
- one-line outcome
- optional hover/click detail

---

Case studies should feel executive and evidence-based.

- light editorial background or dark cards on light background

Each case card should include:

- dark surface or image overlay

## Visual Direction

Insights should feel like an executive publication.

Recommended style:

- light background
- editorial cards
- category chips
- strong titles

Each article card should include:

- image area, 16:9 ratio
- pillar chip
- title
- short excerpt

# Section 8 — About Snapshot

## Visual Direction

This section should humanize the brand without losing executive authority.

Recommended style:

- editorial two-column layout

## Credential Chip Style

Credential chips should be:

- pill-shaped
- 1px border
- small text

## Visual Direction

This section should reinforce public authority.

Recommended style:

- compact dark band or light editorial strip
- talk titles as cards or list items
- CTA to speaking page

---

# Section 10 — AI Maturity Assessment

## Visual Direction

This section should feel like a premium executive diagnostic.

Recommended style:

- dark gradient block
- subtle AI visual details
- clear diagnostic promise
- 3-step readiness preview
- strong CTA

The visual should feel more like an executive assessment than a quiz.

---

## Assessment Preview Style

Use a compact card or panel with three readiness dimensions:

1. Strategy readiness
2. Operating model readiness
3. Adoption and execution readiness

Each dimension can use:

- icon
- label

## Visual Direction

This section should feel quiet, premium and editorial.

Recommended style:

- light or white background
- simple horizontal layout
- minimal form
- clear promise

No pop-up behavior at launch.

## Form Style

Email field:

- large input
- subtle border

- rounded corners matching button system

Button:

- accent background
- clear label

Success state:

- simple confirmation message

---

# Section 12 — Final CTA Band

## Visual Direction

The final CTA should feel decisive and premium.

Recommended style:

- deep navy background
- strong headline
- short supporting copy
- primary CTA

Avoid multiple competing CTAs.

# 6. Component UI Direction

## 6.1 Buttons

Primary button:

- accent background
- white text
- medium font weight
- right arrow icon
- strong hover state

Secondary button:

- transparent
- 1px border
- text color adapts to surface
- hover fill subtle

Text link:

- inline arrow
- accent hover
- underline or arrow movement

---

## 6.2 Cards

General card rules:

- rounded corners: 16px to 24px
- padding: 28px to 36px
- subtle border
- low shadow or no shadow
- hover lift on desktop
- tap feedback on mobile

Card content hierarchy:

1. icon or image
2. label
3. title
4. description
5. CTA link

---

## 6.3 Icons

Icon style:

- thin line icons
- 1.5px stroke
- minimal
- modern SaaS-style

Recommended icon libraries:

- Lucide
- Phosphor

Icon usage:

- service pillars
- assessment dimensions
- navigation support
- article metadata

Avoid:

- colorful icon packs
- filled cartoon icons
- inconsistent icon families

---

## 6.4 Forms

Form style:

- large fields
- strong readability
- subtle borders
- clear focus states
- inline validation
- no unnecessary fields

Forms must feel executive and trustworthy.

---

# 7. Motion UI Direction

## 7.1 Motion Personality

Motion should feel:

- calm
- premium
- subtle
- intelligent
- intentional

Motion should never feel playful, gamified or excessive.

---

## 7.2 Recommended Animations

Hero:

- staggered fade-up for text
- subtle portrait parallax
- slow background glow

Cards:

- lift 4px on hover
- border accent transition
- arrow movement

KPIs:

- count-up once on viewport entry

Framework:

- SVG line draw
- phase hover/click reveal

Navigation:

- glass blur on scroll
- height compression

Newsletter/Form:

- smooth focus state
- success check animation

---

## 7.3 Motion Constraints

Respect:

- prefers-reduced-motion

Avoid:

- scroll hijacking
- constant moving backgrounds
- heavy canvas animations
- large Lottie files
- complex 3D effects

Performance is part of the premium experience.

---

# 9. Accessibility Direction

The UI must meet strong accessibility standards.

Requirements:

- high color contrast
- visible focus states
- keyboard navigation support
- semantic HTML structure
- reduced-motion support
- readable font sizes
- descriptive alt text for images

Avoid relying only on color to communicate meaning.

---

# 10. Performance Direction

The UI must feel fast.

Performance principles:

- optimize all images
- avoid heavy animation libraries unless justified
- keep particle effects lightweight
- lazy-load non-critical media
- use SVGs for simple diagrams
- keep fonts optimized
- avoid unnecessary JavaScript

Premium perception depends on speed.

---

# 11. Static Site Implementation Direction

The UI should be designed for a static site architecture.

Recommended implementation assumptions:

- reusable components
- content-driven sections
- markdown-based Insights
- markdown or JSON-based Case Studies
- optimized static assets
- SEO metadata per page
- GitHub Pages compatibility

The UI should not depend on WordPress behavior or CMS-specific blocks.

---

# 12. UI Approval Checklist

Before moving into frontend development, validate:

- Does the homepage feel premium within the first viewport?
- Does the hero communicate AI Strategy and Enterprise Transformation clearly?
- Does the visual language avoid AI hype?
- Is the accent color used intentionally?
- Are the CTAs visually clear?
- Do cards feel modern and executive?
- Does Insights feel like an executive publication?
- Do case studies feel proof-driven?
- Does the About section feel human but not CV-like?
- Is the AI Maturity Assessment visually premium?
- Does mobile remain clean and clear?
- Can this UI be implemented efficiently as a static site?
