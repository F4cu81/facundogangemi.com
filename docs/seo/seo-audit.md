# SEO Audit — facundogangemi.com
**Date:** 2026-05-12
**Auditor:** Claude Code (automated analysis)
**Scope:** Full site audit — technical SEO, on-page, metadata, headings, images, internal linking, accessibility/UX

---

## Executive Summary

The site is architecturally well-prepared for SEO: Astro generates clean static HTML, canonical URLs are correct, structured data is in place, and the global layout (BaseLayout) handles OG, Twitter Card, hreflang and canonical tags consistently. The visual and content quality is strong for the advisory positioning.

**Critical gaps** that must be resolved before the site can build topical authority:

1. **robots.txt sitemap URL is wrong** — it points to `/sitemap.xml` but the actual generated file is `/sitemap-index.xml`. Google cannot find the sitemap via robots.txt.
2. **Zero published content** — No insights articles, no case studies are live. The entire thought leadership content hub exists as empty architecture.
3. **hreflang targets non-existent /es/ pages** — i18n is configured in `astro.config.mjs`, which causes the sitemap to emit `hreflang` alternate links for Spanish routes that do not exist, creating broken alternates.

The remaining issues are medium or low priority and can be resolved in a structured way after the critical items are addressed.

**Overall rating:** Foundation is solid (7/10). Content strategy has not started. Technical SEO has one critical error and two high-priority issues.

## Technical SEO Findings

### 1. robots.txt — CRITICAL

**Current:**
```
Sitemap: https://facundogangemi.com/sitemap.xml
```

**Actual generated file:**
```
https://facundogangemi.com/sitemap-index.xml
```

`@astrojs/sitemap` generates `sitemap-index.xml` (and `sitemap-0.xml`), not `sitemap.xml`. The URL declared in robots.txt returns a 404 in production. Google Search Console will report this as an error and cannot use the sitemap declaration for crawl guidance.

**Fix:** Change the Sitemap line in `public/robots.txt` to:
```
Sitemap: https://facundogangemi.com/sitemap-index.xml
```

---

### 2. hreflang Configuration — HIGH

`astro.config.mjs` includes:
```js
i18n: {
  defaultLocale: 'en',
  locales: [{ codes: ['en-US'], path: 'en' }, { codes: ['es-ES'], path: 'es' }]
}
```

This causes `@astrojs/sitemap` to generate `<xhtml:link rel="alternate" hreflang="es-ES">` tags in the sitemap pointing to `/es/*` routes. These routes either do not exist or are minimal stubs. Broken hreflang alternates can confuse Googlebot and may be flagged in Search Console.

**Fix:** Remove the `i18n` block from `astro.config.mjs` until Spanish content is ready. Reintroduce it when `/es/` routes are fully implemented.

---

### 3. Sitemap — GOOD (once robots.txt is fixed)

- `@astrojs/sitemap` is correctly installed and configured.
- `site: 'https://facundogangemi.com'` is set correctly.
- `sitemap-index.xml` and `sitemap-0.xml` are generated at build time.
- All public pages appear to be included.
- No accidental `noindex` pages in the current configuration (except draft content which is excluded from collections).

## Accessibility & UX SEO Review

### Verified GOOD:
- Skip navigation link present in BaseLayout (`<a href="#main-content">`)
- Semantic HTML structure (main, nav, footer, article-level sectioning)
- Keyboard navigation — Navbar has visible active/hover states
- Interactive world map markers use `<button type="button">` with `aria-label` — keyboard accessible
- Map markers have `focus-visible` styling with `outline` — keyboard navigable
- Eyebrow labels use `<span>`, not heading tags — correct semantic choice
- `prefers-reduced-motion` respected for marquee animation (paused when preference is set)
- Footer legal links are visible and keyboard accessible
- All `<img>` elements on world map have alt text

### Issues requiring attention:

**1. Map marker focus contrast (Medium)**
The `.map-marker:focus-visible` outline is `2px solid rgba(245, 124, 0, 0.55)`. Orange at 55% opacity on a very dark background may not consistently meet the WCAG 2.1 AA 3:1 contrast ratio for UI component focus indicators. Consider increasing opacity to `1.0` or adding a contrasting outline offset.

**2. Country chip hover state text contrast (Low)**
When `.country-chip.active` is triggered, text color becomes `#F57C00` on a dark background. Verify this meets 4.5:1 contrast ratio. Orange (#F57C00) on near-black (#05070D) typically passes, but confirm against the actual chip background color.

**3. Language switcher UX (Low)**
The ES language toggle in the Navbar is visible but disabled ("coming soon"). If it renders as a clickable link that goes nowhere, this is a UX issue. Verify it has `aria-disabled="true"` and is not a broken link.

**4. Marquee animation (Note)**
Enterprise Contexts marquee pauses on `prefers-reduced-motion`. This is correct. Verify the pause is a CSS `animation-play-state: paused` approach (not JavaScript-dependent), so it works without JS.

### Known limitation — sitemap still includes noindex pages

`@astrojs/sitemap` generates the sitemap from the page file structure, not from robots meta tags. Pages marked `noindex` will still appear in `sitemap-0.xml`. This is not a critical issue (Google can handle noindex pages in sitemaps) but is not ideal. Resolution options for a future phase:
- Use the sitemap `filter` option to explicitly exclude known stub URLs.
- Or accept current behavior until stub pages have real content and noindex is removed.
